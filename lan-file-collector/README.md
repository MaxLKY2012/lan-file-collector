# 局域网文件回收系统

Windows 局域网文件收集服务端，客户端通过浏览器访问 IP 地址，填写用户名并上传文件，服务端自动按用户名分类保存。

## 功能特点

- ✅ **必填用户名**：用户必须填写用户名才能上传
- ✅ **自动分类**：每个用户单独文件夹保存 `保存路径\【用户名】\文件`
- ✅ **同名不覆盖**：同名文件自动重命名 (`file.zip` → `file_1.zip`)
- ✅ **多文件上传**：支持一次上传多个文件
- ✅ **支持大多数文件类型**：文档、压缩包、图片、安装包等都可以
- ✅ **支持中文**：用户名和文件名都支持中文
- ✅ **响应式界面**：手机/电脑浏览器都能用
- ✅ **单文件服务端**：只有一个 `server.py`，简单方便

## 快速开始

### 1. 启动服务

Windows 直接双击打开 `start.bat` 即可：

- 自动检查 Python 环境
- 自动安装依赖
- 提示你输入保存路径（直接回车使用默认 `D:\uploads`）

### 或者命令行启动

```bash
# 安装依赖
pip install -r requirements.txt

# 默认保存位置: D:\uploads
python server.py

# 自定义保存位置
python server.py D:\Your\Custom\Folder
```

### 3. 客户端访问

服务端启动后会输出类似这样:

```
==================================================
 📁 局域网文件回收系统 - 服务端已启动
==================================================

  客户端访问地址:
  http://192.168.1.100:5000

  保存位置:
  D:\uploads

  每个用户独立文件夹:
  D:\uploads\【用户名】\你的文件

==================================================
```

同一局域网下的用户直接在浏览器访问 `http://【服务端IP】:5000` 就可以上传文件了。

## 使用说明

### 服务端
- 默认端口: `5000`
- 默认监听所有网卡 (`0.0.0.0`)，局域网内都可以访问
- 自动创建文件夹，不需要手动建
- 默认保存位置: `D:\uploads`，可在启动时自定义

### 客户端
- 打开浏览器访问服务端IP
- **必须填写用户名**才能上传
- 可以选择多个文件一次上传
- 上传成功会显示提示和文件列表

### 文件类型支持

✅ **支持所有文件类型**，没有任何扩展名限制
- 文档、压缩包、图片、视频、程序、设计文件... 什么都能传

仍然会过滤掉 Windows 文件名不允许的特殊字符 `<>:/\|?*`，保证文件名合法。

## 防火墙说明

- 如果客户端无法访问，检查Windows防火墙，允许Python/端口5000通过
- 如果需要改端口，直接修改 `server.py` 最后的 `port = 5000`
